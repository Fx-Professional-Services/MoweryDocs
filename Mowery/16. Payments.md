
# Payments

## Overview

The **Payment** screen allows staff to track customer payments and manage the payment status. You can search for customers, view payment methods, and mark payments as **Collected**, **Received**, or **Deposited**.

Use this page to ensure all payments move through the correct processing stages.

---

## Open the Payment screen

1. From the main menu, click **Payments**.
2. The Payment page opens and displays a list of customer payment records.

---

## Payment screen layout

The Payment screen contains the following elements:

### Search field

Use the **Search** field at the top of the page to filter payment records.

Example searches:

- Customer name
    
- Partial customer name
    

The results update to show matching records.

---

### Payment table

The payment table displays payment records in rows.

|Column|Description|
|---|---|
|**Customer**|The name of the customer who made the payment.|
|**Payment Method**|The type of payment used (for example: Cash or Check).|
|**Collected**|Indicates the payment has been collected from the customer.|
|**Received**|Indicates the payment has been received by the business.|
|**Deposited**|Indicates the payment has been deposited into the bank.|

Each payment record appears on its own row.

---
# Creating a Collection Report for Payments Received

## Overview

You can generate a collection report to review payments received by technicians. The report provides details such as payment status, date received, and the staff member who received it.

---

## Steps to Create a Collection Report

1. From the **Main Menu**, click **Payments**.
2. Select a payment record to open the **Payment Details** view.
3. On the left side, choose a **Technician** from the list.
4. Click **Collection Report**.

---

## Viewing the Report

- The report displays a list of payment records.
- You can view the following details for each payment:
    - **Received** – whether the payment has been collected
    - **Date Received** – when the payment was collected
    - **Received By** – which technician collected the payment

---

## Summary

The Collection Report allows you to track payments received by technicians, providing clear visibility into collection status, dates, and responsible staff.

# Payment Record Details

## Overview

The **Payment Record Details** screen shows complete information for a specific payment.  
Use this screen to:

- Review customer and billing information
    
- Record collection details
    
- Apply payments to invoices
    
- Track receipt and deposit status
    

Each payment record includes customer details, payment status, collection data, and invoice application information.

Each payment record includes customer details, payment status, collection data, and invoice application information.

---

# Open a Payment Record

1. From the main menu, navigate to the **Payment** page.
    
2. Locate the customer payment in the list.
    
3. Click the payment record.
    

The **Payment Record Details** screen opens.

---

# Screen Layout

The Payment Record Details screen contains several sections.

## Search and Navigation

The top of the screen includes navigation and search tools.

|Tool|Description|
|---|---|
|**List View**|Returns to the list of payment records.|
|**Find**|Search for a specific payment record.|
|**Find All**|Display all payment records.|
|**QuickFind Search**|Filter records by keyword.|
|**Record Navigation**|Move to the first, previous, next, or last payment record.|

---

# Customer Section

The **Customer** section displays the customer associated with the payment.

Fields include:

|Field|Description|
|---|---|
|**First Name**|Customer's first name|
|**Last Name**|Customer's last name|
|**Email**|Customer contact email|
|**Phone**|Customer phone number|

This information is automatically populated from the customer record.

---

# Billing Information

The **Billing Information** section displays the billing address used for the payment.

|Field|Description|
|---|---|
|**Name**|Full billing name|
|**First / Last**|Customer name fields|
|**Billing Address**|Street address|
|**City / State / Zip**|Billing location|

Use this information to confirm the correct billing account.

---

# Status Section

The **Status** section indicates the current processing state of the payment.

|Field|Description|
|---|---|
|**Status**|Current payment stage (for example: Collected).|
|**Notes**|Optional notes about the payment.|

Update the notes field when additional explanation is required.

---

# Collection Section

The **Collection** section records when and how the payment was collected.

|Field|Description|
|---|---|
|**Collected By**|Staff member who collected the payment|
|**Date Collected**|Date the payment was received from the customer|
|**Payment Method**|Type of payment (Cash, Check, etc.)|
|**Check Number**|Check number if payment method is Check|
|**Amount**|Total payment amount|
|**Unapplied Amount**|Portion of the payment not yet applied to invoices|

---

# Invoice Application Table

The table on the right side shows invoices that can receive payment.

|Column|Description|
|---|---|
|**Invoice Number**|Unique invoice identifier|
|**Date**|Invoice creation date|
|**Due Date**|Invoice payment deadline|
|**Invoice Amount**|Total invoice amount|
|**Unpaid Balance**|Remaining balance on the invoice|
|**Applied Amount**|Amount applied from this payment|

Example:

- Invoice: **100026**
    
- Invoice Amount: **$1,237.00**
    
- Applied Amount: **$1,237.00**
    
- Status: **Partially Applied**
    
---

# View Record History

At the bottom of the screen you can see audit information.

|Field|Description|
|---|---|
|**Created By**|User who created the payment record|
|**Created On**|Date and time the record was created|
|**Modified By**|Last user who modified the record|
|**Modified On**|Date and time of the last update|

This information helps track changes to payment records.

# Collection Report (Technician Filter)

## Overview

The **Collection Report** allows you to generate a report of payments collected by a specific technician.

Use this feature to:

- Track payments collected by field technicians
    
- Review technician collection activity
    
- Generate reports for accounting or reconciliation
    

The report is filtered using the **Technician** dropdown in the left panel.

---

# Generate a Collection Report

Follow these steps to create a technician-specific collection report.

1. Locate the **Search Criteria** panel on the left side of the screen.
    
2. Open the **Technician** dropdown.
    
3. Select the technician whose collection data you want to view.
    
4. Click **Collection Report**.
    

The system generates a report showing payments collected by the selected technician.

---

# Search Criteria Panel

The **Search Criteria** panel provides filtering options before running a report.

|Field|Description|
|---|---|
|**Technician**|Dropdown list of available technicians.|
|**Collection Report**|Generates a report using the selected technician filter.|

You must select a technician before generating the report.
