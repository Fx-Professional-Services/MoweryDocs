# 🛠️ Service Calls

This guide explains how to use key features of the **Service Call** module in Mowery FileMaker. All actions described here are accessible from the **Service Calls** main menu.

A **Service Call** is a scheduled visit for installation, repair, maintenance, or inspection work. It may involve HVAC, plumbing, or other services provided to a customer.

These tools help you:

- Assign technicians using the Dispatch Board  
- Send technician details to customers by text message  
- Provide customers with a secure Technician Profile Page  
- Print a job-specific parts list  
- Manage technician and staff records  
- Understand system auto-logout behavior  
- Organize Categories and Subcategories for product assignments

## Assign Technicians Using the Dispatch Board

The **HVAC Schedule Board** helps dispatchers assign technicians to service calls quickly.

### How to Assign a Technician

1. Click **Service Calls** from the main menu.  
2. Select **Schedule Board**.  
3. Unassigned HVAC calls appear in the left column.  
4. Click a service call to open the details.  
5. In **Step 2: Assign Technician**, click **New**.  
6. Select a technician from the dropdown list.  

> Only preconfigured technicians appear in the list.  

### What is HVAC?

**HVAC** means **Heating, Ventilation, and Air Conditioning**. These calls are managed separately using the HVAC Dispatch Board.

## Send a Text Message with Technician Details

You can send customers a personalized text message with technician and appointment details using Twilio.

### Requirements

- The **Master Property List** phone number is:  
  - Valid  
  - Marked as **Preferred**  

- In the **Service Call** record:  
  - **Accept Text Message** is checked  

- Phone number uses valid 10-digit U.S. format  

### How to Send a Text Message

1. Click **Service Calls**.  
2. Open the service call record.  
3. In the **Contact** section:  
   - Confirm **Text** is selected  
   - Ensure the number is valid and marked **Preferred**  
   - Check **Accept Text Message**  
4. Click the **Send Text Message** icon.  
5. Wait for the confirmation or error message.  

### Message Contents

- Technician name(s)  
- Customer address  
- Secure link to the **Technician Profile Page**  
- Technician introduction  

**Example message:**  

Hello [Customer Name], [Technician Name(s)] from Mowery is on the way to [Customer Address].  
Click here for more details: [Technician Profile Webpage Link]  
Hi, my name is [Technician Name]. I’ll be your technician today. [Short Technician Bio]  
If you have any questions, I’m here to help. Thank you for choosing Mowery!


### Template Variables

| Variable                        | Description                            |
|----------------------------------|----------------------------------------|
| `[Customer Name]`               | Full name of the customer              |
| `[Technician Name(s)]`          | Name(s) of assigned technician(s)      |
| `[Customer Address]`            | Service location                       |
| `[Technician Bio]`              | Technician's short introduction        |
| `[Technician Profile Webpage Link]` | Secure link to technician profile   |

### Error Messages

| Condition                                | Message                                              |
|------------------------------------------|------------------------------------------------------|
| No preferred number                      | A preferred phone number is required to send a message. |
| Number missing, checkbox checked         | Cell number is required if Accept Text Message is checked. |
| Invalid number                           | Invalid US Number.                                   |
| Twilio delivery failure                  | Failed to deliver.                                   |
| Other Twilio errors                      | Message shown directly from Twilio                   |

## Technician Profile Page for Customers

When customers receive a service notification text, it includes a secure link to the **Technician Profile Page**.

### What Customers See

**Service Information:**  
- Status (Scheduled, In Progress, or Completed)  
- Appointment time  
- Service description  
- Service type (e.g., HVAC, Electrical)  

**Technician Details:**  
- Technician name(s)  
- Photos  
- Short biography (if available)  

### Security and Access

- Each link is unique and secure  
- No login required  
- Customers cannot access other profiles  
- Mobile-friendly design  

This improves customer confidence and transparency before a technician arrives.

## Print a Parts List from a Service Call

You can print a job-specific parts list using **Print to Parts**.

### How to Print to Parts

1. Click **Service Calls**.  
2. Select a service call from the rightmost column.  
3. Scroll to the bottom-right corner.  
4. Click **Print to Parts**:  
   - A print layout opens  
   - Ignore "Printer not found" message if it appears  
5. The **Printed to Parts** field updates automatically with:  
   - Dispatcher’s account name  
   - Date and time  

### Default Printer Behavior

- The system sends the job to the **default parts printer**  
- No manual selection required  

**Benefits:** Faster printing, fewer manual steps, improved workflow tracking.

## View and Manage Staff Records

Manage employee records from the **Service Calls** module.

### View the Staff List

1. Click **Service Calls**.  
2. Select **Staff**.  
3. The **Staff List** layout displays all employees (read-only).  

### Edit an Employee Record

1. In **Staff List**, click the **Form View** button (number to the left of the employee’s name).  
2. The **FileMaker Access** layout opens.  
3. Edit employee details. Changes save automatically.  

### Add a New Employee

1. In **FileMaker Access**, click **New**.  
2. Enter the employee’s details. Changes save automatically.  

### Delete an Employee Record

- From **Staff List**, click **Delete Account** to remove the FileMaker account.  
- Click the red **(X)** to remove the employee record and history.  
- From **FileMaker Access**, click the **(X)** to delete the employee. Confirm when prompted.

## View Product Price History

Track price updates in the **Price History Tracking** panel.

### 🕘 Steps

1. Go to **System Defaults → Products**.
    
2. Select a product.
    
3. Find the **Price History Tracking** panel (upper-right corner).
    

### What You’ll See

|Field|Description|
|---|---|
|**Effective Date**|Date when pricing was updated|
|**Revision**|Version number|
|**Standard Rate**|Regular sale price|
|**Value Rate**|Value-based sale price|
|**Add-On Standard**|Standard price for add-ons|
|**Add-On Value**|Value-based price for add-ons|
|**Commercial**|Commercial pricing|

> **Note:** All pricing updates are automatically logged for auditing.
