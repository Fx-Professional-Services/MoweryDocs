The **System Defaults** section lets administrators configure core settings in the Mowery system. Use it to:

- Create and manage **product catalogs** (price books), including pricing and tax settings.
    
- Maintain **product pricing** and track historical changes.
    
- Configure **system-wide dropdown values**, such as employee lists and categories.

# Catalog Management

A **catalog** (or **price book**) defines product pricing effective from a specific date. Catalogs support:

- Duplication
    
- Searching
    
- Manual reordering via **Task Codes**
    
- Multi-category product assignments
---
### Create a Catalog

1. Go to **System Defaults → Catalog**.
    
2. Click **New Catalog**.
    
3. Enter the following information:
    
    - **Effective Date** (must be unique)
        
    - **Title**
        
    - **Revision**
        
    - Other required fields
        
4. Click **Add Catalog**.
    
5. Confirm when prompted.
    
6. Select the catalog name to open it.
    
7. Verify the number of catalog items matches your active product list.
    

> **Tip:** In Mowery, **Catalog** and **Price Book** mean the same thing.

Catalog items are generated as follows:

- One item per active product, **or**
    
- Multiple items if a product has several category/type associations

3. In the **Category/Subcategory** section, review assigned rows. Each row represents an additional category linked to the product.

## Manage Task Codes (Reorder Items)

**Task Codes** control the sequence of catalog items. You can adjust task codes to reorder items manually.

### Task Code Basic

- Define item order within categories/subcategories.
- Assigned automatically for new items.
- Catalog order updates immediately after edits.

### Reorder Items

1. Go to **System Defaults → Catalog**.
2. Click the catalog title to open it.
3. Find the item to reorder.
4. Click the item’s **Code** field.
5. Enter a new numeric task code.
6. The catalog updates automatically.

> **Tip**: Use gaps between task codes (e.g., 10, 20, 30) to simplify inserting new items later.

### Task Code Behavior by Category

Task code sequencing depends on **Category Code** and **Subcategory Code**:

|Category|Subcategory|Task Code Sequence|
|---|---|---|
|A|A|1, 2, 3...|
|A|B|1, 2...|
|A|A|4, 5...|

**Behavior Notes**:

- Task codes continue within the same category and subcategory.
- Changing either resets the sequence to **1** for that group.
- Returning to a previous group resumes the task code sequence.

This structure ensures catalog order stays organized and easy to manage.

# Category

Categories and subcategories organize products for catalogs, pricing, and reporting.

### Create and Manage Categories

1. Go to **System Defaults → Category**.
    
2. Click **New Category**.
    
3. Enter **Category Name**.
    
4. Select **Type**: `HVAC` or `Plumbing`.
    

> **Note:** Type exists only in the **Category** table. It distinguishes categories with the same name.

### Duplicate Category Names

You can reuse a name if **Type** differs (e.g., `Miscellaneous (HVAC)` vs. `Miscellaneous (Plumbing)`).

### Assign Subcategories to Categories

1. Go to **System Defaults → Sub Category**.
    
2. Select a subcategory.
    
3. In the **Category** dropdown, select the parent category.
    

> **Note:** Subcategories inherit the Type from the parent category.

### Assign Products to a Subcategory

- Every product must have a subcategory.
    
- If no specific subcategory applies, assign to `Miscellaneous (HVAC)` or `Miscellaneous (Plumbing)`.

### Prevent Data Loss When Deleting Categories or Subcategories 

When managing products, deleting a category or unassigning a product from a category or subcategory **does not affect pricing data**.

#### Key Points

- You can safely delete categories or subcategories in the **Item/Product screen**.
- Product pricing remains intact after changes.

> ⚠️ **Important:** This behavior applies **only in the Item/Product screen**. Category and subcategory settings in the **Developer Settings layout** behave differently and are not covered here.

# Products

The **Products** section stores pricing and details for all items. Pricing changes are logged with effective dates and history tracking.

---

## View Product Details

1. Go to **System Defaults → Products**.
    
2. Select a product record.
    
3. Click the leftmost button to open **Product Info** and **Product Price Attributes & Preview**.
    

---

## Adjust Automatic Rounding

In the **Product Price Attributes & Preview** section, you can control automatic rounding for each rate type.

### To change rounding behavior

- Use the following checkboxes on the left panel:
    
    - **Automatic Round Off – Standard**
        
    - **Automatic Round Off – Value**
        
    - **Automatic Round Off – Commercial**
        
- **Checked** — The system automatically rounds the computed price.
    
- **Unchecked** — The system keeps the exact calculated price.
    

> **Note:** Rounding is applied per rate type and immediately updates the preview prices.
    

---

### Automatic Price Rounding

The system can round prices automatically to make them easier to read. Rounding affects **only what is displayed**; all calculations in the system still use the exact values with cents.

#### How Rounding Works by Rate Type

- **Standard Rate**: Rounded to the nearest whole dollar.  
    Example: $349.25 → $349
    
- **Value Rate**: Rounded to the nearest whole dollar.  
    Example: $289.80 → $290
    
- **Add-On Standard**: Rounded to the nearest whole dollar.  
    Example: $59.49 → $59
    
- **Add-On Value**: Rounded to the nearest whole dollar.  
    Example: $45.51 → $46
    
- **Commercial Rate**: Rounded to two decimal places.  
    Example: $89.236 → $89.24, $89.234 → $89.23
    

#### Rounding Rules

- Values ending in **.50 or higher round up**
    
- Values below **.50 round down**
    
- Changes are **immediate in preview prices** but do **not affect backend accounting**
    
- All pricing changes are **logged in the Price History Tracking panel**

## Edit Product Pricing

Manually update product pricing in the **Current Pricing** section. Updates are tracked in the **Price History Tracking** panel.

### Steps

1. Go to **System Defaults → Products**.
    
2. Select a product.
    
    > _Tip:_ Use the filter to select a type (e.g., HVAC, Plumbing).
    
3. Click the **leftmost box** of the row to open the product record.
    
4. In the **Product Price Attributes**, you can change the **Cost** and **Est. Hours**.
    
    > **Note:** Changes to **Cost** and **Est. Hours** will **not immediately reflect** in the **Current Pricing** section. Create a **new catalog** to see the updated prices.
    
5. To update a product’s active price, edit the **Standard**, **Value**, or **Commercial Rate** directly in the **Current Pricing** section.
    
    > **Note:** Changes made in **Current Pricing** reflect **immediately** in the record.
    
6. When a price changes, a **reset icon (🔄)** appears beside the modified field.  
    Click it to **revert to the default value**.
    
## Additional Information

- **Price Book Updates**  
    All price changes are automatically reflected in the current price book.
    
- **$0 Commercial Products**  
    Products marked with **$0 Commercial** retain a **$0 commercial rate** when creating, duplicating, or updating catalogs.

## Reorder items using **Sort Order**

Use the **Sort Order** button to change the display order of product descriptions by editing their position numbers.

---

### Reorder items by number

1. Click **System Preferences** > **Products**.
2. Select a product record.
3. In the lower-right panel, locate the list of product descriptions.
4. In the **Sort Order** column, click the number next to the item you want to move.
5. Enter a new number:
    - To move the item up, enter a smaller number (for example, change `3` to `1`).
    - To move the item down, enter a larger number (for example, change `1` to `4`).
6. Repeat for other items as needed.
7. Click **Sort Order** to apply the changes.

The system updates the list based on the new numbers.

---
### Reserialize sort order

Use this option to reset numbering after multiple edits.

1. Click **Sort Order**.
2. Click **Reserialize sort order**.

The system renumbers items sequentially (1, 2, 3, …) based on their current order.

---

### Verify changes

- Confirm that items appear in the expected order.
- Verify that numbering is sequential and matches the display order.

---

### Tips

- Use whole numbers (1, 2, 3) to keep ordering consistent.
- Avoid duplicate numbers.
- After several edits, use **Reserialize sort order** to standardize numbering.

# Updating Product Pricing After Labor Rate Changes

## Overview

When you change labor rates in **Developer Settings**:

- **Product Price Preview** immediately shows the updated rates.
- **Current Pricing** does **not** automatically update. To synchronize Current Pricing with Product Price Preview, you must create a **new catalog** with an appropriate effective date.

---

## When to Use This Guide

Use this procedure when:

- You updated HVAC, Plumbing, or Value Rates in Developer Settings
- You want Current Pricing to match Product Price Preview
- You need accurate pricing for active products

---

## Before You Begin

- Confirm labor rates are updated and saved in **Developer Settings**
- Decide on the **effective date** for the new catalog

---

## Step 1: Update Labor Rates

1. From the main menu, click **System Defaults** → **Developer Settings**
2. Update the following fields as needed:
    - HVAC Labor Rate
    - Plumbing Labor Rate
    - Value Rate
3. Save your changes

> **Note:** Product Price Preview will reflect these changes immediately, even before creating a new catalog.

---

## Step 2: Create a New Catalog

1. From the main menu, navigate to **System Defaults → Catalogs**
2. Click **Create New Catalog**
3. Set the **Effective Date** to the current date or a future date
4. Save the catalog
5. Wait for the system to confirm that the new catalog has been created

> **Important:** If you do **not** create a new catalog, Current Pricing will remain outdated and will **not** match Product Price Preview.

---

## Step 3: Verify Pricing

1. From the main menu, go to **System Defaults → Products**
2. Click the leftmost button to open a product
3. Navigate to **Product Price Attributes & Preview**
4. Compare **Product Price Preview** with **Current Pricing** (bottom-right corner)

### Expected Result

- Product Price Preview shows updated labor rates immediately
- Current Pricing matches Product Price Preview **only** if a new catalog was created with the correct effective date

---

## Troubleshooting

- **Current Pricing does not match Product Price Preview:**
    - Confirm a new catalog exists with an effective date on or after the labor rate change
    - Verify labor rates were saved in Developer Settings
- **Preview shows updated rates but Current Pricing is outdated:**
    - This is expected if no new catalog was created

---

## Summary

- Product Price Preview updates immediately after labor rate changes
- Current Pricing updates **only** when a new catalog is created
- Always create a new catalog with an effective date on or after the labor rate change to synchronize Product Price Preview and Current Pricing

# Updating Labor Rates Directly in a Catalog

## Overview

If you update labor rates directly in an existing catalog, **Current Pricing will reflect the changes immediately** — **but only if the catalog’s effective date is today**.

This allows you to synchronize Product Price Preview and Current Pricing without creating a new catalog.

---

## When to Use This

Use this procedure when:

- You need Current Pricing to match Product Price Preview immediately
- You are updating rates for an existing catalog whose effective date is today

---

## Steps

1. From the main menu, navigate to **System Defaults → Catalogs**
2. Select the catalog by clicking the **leftmost button**
3. Update the following labor rates as needed:
    - HVAC Labor Rate
    - Plumbing Labor Rate
    - Value Rate
4. Click **Update Price**

> **Important:** Changes only update Current Pricing immediately if the catalog’s effective date is **today**.

---

## Verification

1. Go to **System Defaults → Products**
2. Open a product
3. Navigate to **Product Price Attributes & Preview**
4. Compare **Product Price Preview** with **Current Pricing**

> **Expected Result:** Current Pricing matches Product Price Preview for the catalog edited today.

---

## Notes

- If the catalog’s effective date is in the past or future (not today), Current Pricing will **not** update immediately.
- Product Price Preview always reflects the changes immediately, regardless of catalog date.

# Preserve List Position When Returning to List View

When working with long product or catalog lists, the system preserves your **scroll position** when you return to the list using the **List View** button. This prevents the list from resetting to the top and allows you to continue working where you left off.

The list does **not automatically re-sort or jump** unless you explicitly apply sorting or filtering.

---

## Return to the Same Position in the List

When you open a product from the list and return using the **List View** button:

- The system restores the **same scroll position**.
    
- The **same product remains visible** in the list.
    
- The item appears in the **same relative position** on the screen.
    
- The list **does not reset to the top**.
    

This allows you to continue reviewing or editing items without searching for your previous location.

---

## Steps

1. Open a **Catalog** or **Product List**.
    
2. Scroll to a product in the list.
    
3. Click the product to open its details.
    
4. Make any changes if needed.
    
5. Click **List View** to return to the list.
    

Result:

- The list returns to the **same position** where you were previously working.
    
- The product you opened remains **visible in the list**.
    

---

## When the List Position Changes

The list position changes only when you perform actions that modify the list order, such as:

- Sorting a column
    
- Applying filters
    
- Refreshing the list with new criteria
    

If none of these actions occur, the system preserves your previous position in the list.

# Terminology Reference

| Term                                             | Description                                                                                                            |
| ------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------- |
| **Catalog / Price Book**                         | A list of product prices that applies starting on a set date. Includes tax and labor rules.                            |
| **Effective Date**                               | The day a catalog’s prices start. Each catalog needs its own date.                                                     |
| **Task Code**                                    | A number that sets the order of items in a catalog (e.g., 10, 20, 30). Spacing makes it easier to add new items later. |
| **Standard Rate**                                | The regular price shown to most customers.                                                                             |
| **Value Rate**                                   | A lower price, often used for customers with agreements or promotions.                                                 |
| **Commercial Rate**                              | Pricing for business or large projects.                                                                                |
| **Manual Override**                              | A custom price typed in by an admin that replaces the system’s price.                                                  |
| **Flat Rate**                                    | One fixed cost for a job, no matter how long it takes.                                                                 |
| **Add-On Rate**                                  | An extra charge for additional services.                                                                               |
| **Mobile App SA Add-On Discount**                | A discount when a Service Agreement is purchased in the mobile app.                                                    |
| **Price History**                                | A record of all past price changes, including who made them and when.                                                  |
| **Update Prices Except Manually Modified Ones**  | Updates only system-calculated prices. Manual entries stay the same.                                                   |
| **Update All Prices (Overwrite Manual Changes)** | Updates every price, even manual ones.                                                                                 |
| **FileMaker Access**                             | The screen used to manage system users and employee lists.                                                             |
