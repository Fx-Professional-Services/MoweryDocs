# System Defaults: Catalogs, Products, and FileMaker Access

The **System Defaults** section lets administrators configure core settings in the Mowery system. Use it to:

- Create and manage **product catalogs** (price books), including pricing and tax settings.
    
- Maintain **product pricing** and track historical changes.
    
- Configure **system-wide dropdown values**, such as employee lists and categories.
    

> 🔒 Only users with administrative access can view or edit this section.

# Catalog Management

A **catalog** (or **price book**) defines product pricing effective from a specific date. Catalogs support:

- Duplication
    
- Searching
    
- Manual reordering via **Task Codes**
    
- Multi-category product assignments
---
### Create a Catalog

1. Go to **System Defaults → Catalog**.
    
2. Click **New Catalog**.
    
3. Enter the following information:
    
    - **Effective Date** (must be unique)
        
    - **Title**
        
    - **Revision**
        
    - Other required fields
        
4. Click **Add Catalog**.
    
5. Confirm when prompted.
    
6. Select the catalog name to open it.
    
7. Verify the number of catalog items matches your active product list.
    

> **Tip:** In Mowery, **Catalog** and **Price Book** mean the same thing.

### How the Effective Date Works

Catalog items are generated as follows:

- One item per active product, **or**
    
- Multiple items if a product has several category/type associations
    

**Example:** A product in three categories generates three catalog items.

### Verify Catalog Items Match Active Products

1. Go to **System Defaults → Products → View → Find Mode**.
    
2. Check the **Active** field. Tick the box so that a checkmark (✓) appears. Check the **Active** field.
    
3. Click **Requests → Perform Find**.
    
4. Compare the total record count with your catalog item count.

### Find All Active Products

1. From the **main menu**, go to **System Defaults → Products**.
2. Click **View → Find Mode**.
3. Check the **Active** field (should be marked with a checkmark or “X”).
4. Click **Requests → Perform Find**.
5. Check the total record count in the upper-right corner.

Compare this total with the number of items in your catalog.

**Check Category Associations**

1. Go to **System Defaults → Products**.
    
2. Open a product record.
    
3. In the **Category/Subcategory** section, review assigned rows. Each row represents an additional category linked to the product.

## Duplicate a Catalog 

Duplicating a catalog preserves **manual edits**.

1. Go to **System Defaults → Catalog**.
    
2. Select a catalog.
    
3. Click **Duplicate Catalog**.
    
4. Update details if needed.
    
5. Confirm duplication.
    
6. Open the new catalog and verify:
    
    - Item count matches the original
        
    - Manual edits are preserved
        

> **Tip:** Contact your administrator if edits are lost.

## Manage Task Codes (Reorder Items)

**Task Codes** control the sequence of catalog items. You can adjust task codes to reorder items manually.

### Task Code Basic

- Define item order within categories/subcategories.
- Assigned automatically for new items.
- Catalog order updates immediately after edits.

### Reorder Items

1. Go to **System Defaults → Catalog**.
2. Click the catalog title to open it.
3. Find the item to reorder.
4. Click the item’s **Code** field.
5. Enter a new numeric task code.
6. The catalog updates automatically.

> **Tip**: Use gaps between task codes (e.g., 10, 20, 30) to simplify inserting new items later.

### Task Code Behavior by Category

Task code sequencing depends on **Category Code** and **Subcategory Code**:

|Category|Subcategory|Task Code Sequence|
|---|---|---|
|A|A|1, 2, 3...|
|A|B|1, 2...|
|A|A|4, 5...|

**Behavior Notes**:

- Task codes continue within the same category and subcategory.
- Changing either resets the sequence to **1** for that group.
- Returning to a previous group resumes the task code sequence.

This structure ensures catalog order stays organized and easy to manage.

# Category

Categories and subcategories organize products for catalogs, pricing, and reporting.

### Create and Manage Categories

1. Go to **System Defaults → Category**.
    
2. Click **New Category**.
    
3. Enter **Category Name**.
    
4. Select **Type**: `HVAC` or `Plumbing`.
    

> **Note:** Type exists only in the **Category** table. It distinguishes categories with the same name.

### Duplicate Category Names

You can reuse a name if **Type** differs (e.g., `Miscellaneous (HVAC)` vs. `Miscellaneous (Plumbing)`).

### Assign Subcategories to Categories

1. Go to **System Defaults → Sub Category**.
    
2. Select a subcategory.
    
3. In the **Category** dropdown, select the parent category.
    

> **Note:** Subcategories inherit the Type from the parent category.

### Assign Products to a Subcategory

- Every product must have a subcategory.
    
- If no specific subcategory applies, assign to `Miscellaneous (HVAC)` or `Miscellaneous (Plumbing)`.

### Prevent Data Loss When Deleting Categories or Subcategories

- Deleting a category or unassigning a product from a category/subcategory **does not affect pricing data**.
    
- Users can safely remove categories or update assignments without losing product pricing.

# # Products

The **Products** section stores pricing and details for all items. Pricing changes are logged with effective dates and history tracking.

---

## View Product Details

1. Go to **System Defaults → Products**.
    
2. Select a product record.
    
3. Click the leftmost button to open **Product Info** and **Product Price Attributes & Preview**.
    

---

## Adjust Automatic Rounding

In the **Product Price Attributes & Preview** section, you can control automatic rounding for each rate type.

### To change rounding behavior

- Use the following checkboxes on the left panel:
    
    - **Automatic Round Off – Standard**
        
    - **Automatic Round Off – Value**
        
    - **Automatic Round Off – Commercial**
        
- **Checked** — The system automatically rounds the computed price.
    
- **Unchecked** — The system keeps the exact calculated price.
    

> **Note:** Rounding is applied per rate type and immediately updates the preview prices.
    

---

### Automatic Price Rounding

The system can round prices automatically to make them easier to read. Rounding affects **only what is displayed**; all calculations in the system still use the exact values with cents.

#### How Rounding Works by Rate Type

- **Standard Rate**: Rounded to the nearest whole dollar.  
    Example: $349.25 → $349
    
- **Value Rate**: Rounded to the nearest whole dollar.  
    Example: $289.80 → $290
    
- **Add-On Standard**: Rounded to the nearest whole dollar.  
    Example: $59.49 → $59
    
- **Add-On Value**: Rounded to the nearest whole dollar.  
    Example: $45.51 → $46
    
- **Commercial Rate**: Rounded to two decimal places.  
    Example: $89.236 → $89.24, $89.234 → $89.23
    

#### Rounding Rules

- Values ending in **.50 or higher round up**
    
- Values below **.50 round down**
    
- Changes are **immediate in preview prices** but do **not affect backend accounting**
    
- All pricing changes are **logged in the Price History Tracking panel**

Details about how pricing is calculated.

|Field|Description|

## Edit Product Pricing

Manually update product pricing in the **Current Pricing** section. Updates are tracked in the **Price History Tracking** panel.

### Steps

1. Go to **System Defaults → Products**.
    
2. Select a product.
    
    > _Tip:_ Use the filter to select a type (e.g., HVAC, Plumbing).
    
3. Click the **leftmost box** of the row to open the product record.
    
4. In the **Product Price Attributes**, you can change the **Cost** and **Est. Hours**.
    
    > **Note:** Changes to **Cost** and **Est. Hours** will **not immediately reflect** in the **Current Pricing** section. Create a **new catalog** to see the updated prices.
    
5. To update a product’s active price, edit the **Standard**, **Value**, or **Commercial Rate** directly in the **Current Pricing** section.
    
    > **Note:** Changes made in **Current Pricing** reflect **immediately** in the record.
    
6. When a price changes, a **reset icon (🔄)** appears beside the modified field.  
    Click it to **revert to the default value**.
    

> **Reminder:** Price changes are automatically reflected in the current price book.

### Delete from FileMaker Access

1. From the main menu, click **System Defaults**.
    
2. In the upper-right corner, click **FileMaker Access**.
    
3. In the **FileMaker Access** layout, click **(X)** next to the employee’s name.
    
4. To archive the employee’s FileMaker account instead, click **Archive Account**.
    
5. Confirm the deletion or archiving when prompted.
    

---

### Archive vs. Delete

|Action|What It Does|What It Does Not Do|
|---|---|---|
|**Archive Account**|Removes the account from FileMaker’s **Manage Security** dialog.|Does not remove the employee from the **Staff List**. Historical records remain intact.|
|**Delete (X)**|Removes the employee from the **Staff List** view.|Does not remove the associated FileMaker Security account.|

⚠️ **Important**  
Archiving an account does not delete the employee record or historical data. Service calls, dispatch logs, and customer visit records continue to display the employee’s name even after access is removed.

**Best Practice: 
- Use **Archive Account** when you need to remove or deactivate a user’s access.
    
- Use **Delete (X)** only when you need to remove an employee record from the list but keep their FileMaker account active.

# Terminology Reference

| Term                                             | Description                                                                                                            |
| ------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------- |
| **Catalog / Price Book**                         | A list of product prices that applies starting on a set date. Includes tax and labor rules.                            |
| **Effective Date**                               | The day a catalog’s prices start. Each catalog needs its own date.                                                     |
| **Task Code**                                    | A number that sets the order of items in a catalog (e.g., 10, 20, 30). Spacing makes it easier to add new items later. |
| **Standard Rate**                                | The regular price shown to most customers.                                                                             |
| **Value Rate**                                   | A lower price, often used for customers with agreements or promotions.                                                 |
| **Commercial Rate**                              | Pricing for business or large projects.                                                                                |
| **Manual Override**                              | A custom price typed in by an admin that replaces the system’s price.                                                  |
| **Flat Rate**                                    | One fixed cost for a job, no matter how long it takes.                                                                 |
| **Add-On Rate**                                  | An extra charge for additional services.                                                                               |
| **Mobile App SA Add-On Discount**                | A discount when a Service Agreement is purchased in the mobile app.                                                    |
| **Price History**                                | A record of all past price changes, including who made them and when.                                                  |
| **Update Prices Except Manually Modified Ones**  | Updates only system-calculated prices. Manual entries stay the same.                                                   |
| **Update All Prices (Overwrite Manual Changes)** | Updates every price, even manual ones.                                                                                 |
| **FileMaker Access**                             | The screen used to manage system users and employee lists.                                                             |
